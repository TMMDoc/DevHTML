GitHub Workflow

Git, being a distributed version control system, allows everyone to work on the documentation project.

To understand the documentation workflow, it is required to have a basic understanding of GitHub workflow and git commands used.

Git commands used

Following are the basic git commands used in our documentation process.

  • git init - To create a new git repository.

  • git status - Shows the state of the working directory, staging area, tracked and untracked files.

  • git add - To add changes from the working directory to the staging area.

  • git commit - To save all staged changes, along with a brief description from the user.

  • git pull - To fetch and merge the changes which are present in the remote repository to the local repository.

  • git push - To upload local repository content to a remote repository.

  • git branch - To work on a different parts of a project without impacting the Main branch.

  • git switch - To switch between the branches.

  • git log - To view the history of commited changes within a git repository.

  • git config --global --edit - To edit the configuration file.

  • git restore - To unstage or even discard uncommitted local changes.

  • git diff - To shows the differences between various states of a repository.
