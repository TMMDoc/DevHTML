Workflow for collaboration with documentation

Product Owners

Product owners are responsible for their own product documentation. Product owners should regularly check their documentation to ensure it is complete and readable. Product owners must evaluate the release notes, open a case in Fogbugz, establish priorities, and notify the documentation team.

  • Minor updates (like updating screen shots) don’t need a lengthy description. Just create a Fogbugz case for Documentation with reference to the relevant release note.

  • Major updates and new features require Product Owners to define the substance of the documentation. It is needed to include high-quality images (.png files) with the input files. Provide the input in asciidoc format (if possible) or in a basic word document for the documentation team to work with.

  • Minute updates can be added directly in the respective product’s GitHub repository.

Create a Fogbugz case to inform the documentation team about this minor update.

Procedure for minor updates in GitHub:

  1. Navigate to the file you want to edit.

  2. Click on Edit this file icon and edit your changes.

  1. After editing, click Commit changes…​ to create a new branch with a commit message.

  1. Type the commit message, add description, select Creating a new branch for this commit, and type the branch name.

  1. Click Propose changes to create a pull request.

  1. Goto the Pull Request and assign it to documentation team member.

  2. After the documentation team has reviewed and merged the changes into the main branch, close the Fogbugz case.

Documentation Team

The documentation is responsible for

  • Updating the documentation according the Fogbugz cases created by the Product Owners.

  • Work on major updates (conver input file to asciidoc format), commit changes and assign pull request for review.

Significant changes to document folder structure (Under Development)

Wolf will lend his support on creating macros to:

  • set up new base structures for new documents by simple text files.

  • restructure documents (move folders and pictures).

  • convert existing lengthy word documents via pandoc.

Feel free to contact Wolf to avoid a lot of manual labor.
