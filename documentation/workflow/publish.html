Publish

Document publication is a one-time event. If the Auto-Deploy feature is enabled when using Render to create a static site, Render will immediately publish the changes as soon as content is pushed into the HTML repository.

The main branch of the HTML repository should be linked to the respective static site.

Following are the steps to create a static site:

  1. Sign in into Render using GitHub account.

  2. Click New + and select Static Site to create a static website.

  3. Connect a GitHub repository that you wish to deploy.

    1. Search for <TitleBookHTML> repo.

  4. Fill in the fields in the following format:

    Name = <TitleBookHTML>

    Branch = main

    Root directory = ./

    Build command =

    Publish directory = .

  5. Click Create Static site.

  6. Wait for few seconds for the Render to host your site.

    1. You should see the site Live with latest commit message.

  7. Click the url to do a final review.

    The URL will be created only once per document and remains the same.

  8. Goto, Render Dashboard and select the <TitleBookHTML>.

  9. Navigate to Settings → Build & Deploy → Auto-Deploy and edit it to Yes.
