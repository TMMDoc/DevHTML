Review

  • To review changes in GitHub we need a Pull Request in place.

  • A typical PR page is shown below:

  • Click the files changed tab to view the change difference. On the left side the user can see the source and on the right side is the changes introduced.

  • These changes are listed file wise. Mostly we use .adoc and .png file format to create our documents.

  • By default the user can see the diff in code view .

This gives a detailed view of added and deleted contents from the source file.

  • To view the changes in the rendered output, click on the display the rich diff icon.

  • To add review comment, click the icon next to code line number.

  • To comment directly on a file, click the icon and type your comments.

  • Once you are done with the review, click the submit review button.
