Repositories for Documentation

  1. Create/identify repositories, one for AsciiDoc files and another one for HTML files.

Contact "Metamation GitHub Owner" to create new repositories.

  1. Make sure precommit hook is deployed.

This is to make sure no one is allowed to work or push code directly into main branch.

  1. Add collaborators, reviewers and approvers to <TitleBook> repository.

  2. Add .gitattributes and .gitignore files.

Folder structure

  1. Analyze the topics and chapters required to create a document. This will make easier for us to create a document skeleton.

  2. Create .adoc files for each topic using MS Visual Studio Code. Rearrange the topics if required to have a meaningful chapter.

  3. Make sure images are placed in the respective img folders and check for the naming convention.

Render is highly case sensitive. Make sure you enter the exact image name under image syntax. Also, make sure to save all images in .png format. To avoid confusion, keep the image names entirely in lower-case.

  1. Create content.txt and manifest.txt files for Bento to compile.

  2. Each folder and sub folder represents a chapter and sub chapter respectively.

  3. Every folder and sub folder must have a img folder for images, logos, icons and contents folder for ToC and index.adoc file for home page and an additional chapter or sub chapter .adoc files.

How to create .adoc file

  1. Open VS Code and press ctrl+shift+E to open Explorer.

  2. Click Open Folder, locate the <TitleBook> folder path and click Select Folder.

  3. Hover the mouse pointer over the Explorer panel to see New file and New Folder…​ option.

  1. Click new file, type index.adoc and press Enter. NOTE: You have created your first .adoc file.

  2. Now create the document structure, chapters and sub-chapters.

Set the changes for Review

  1. Goto <TitleBook> folder and initiate CMD.

  2. Type git st to see list of files modified/added/removed.

  3. Type git add . to move all the files to staging area.

  4. Type git commit -m "message" to commit all the changes.

  5. Type git push to push all the files to GitHub repository.

This is to make sure every one can collaborate in document creation process.

  1. Goto <TitleBookHTML> folder and initiate CMD.

  2. Repeat steps 2 to 5.

Only the document owner is responsible for pushing the HTML folder to GitHub.
Every time you update a document (edit a .adoc files) after a git push, make sure you delete the <TitleBookHTML> folder. Clone the <TitleBookHTML> repository freshly and then run bento from the <TitleBook> folder.
Missing this step will make differences in files between <TitleBookHTML> GitHub repo files and <TitleBookHTML> local files.

GitHub

  • Go to GitHub <TitleBook> repo and click Pull Request.

  • In the Pull request page, click New pull request option to compare and review between different branches and commits.

  • Select the branch name and commit details.

  • Leave a short comment about the pull request and click create pull request.

  • Add reveiwer details and set the pull request for review.

The reviewer can either approve and merge the pull request or set it back for rework with comments.
