BT Apply Changes

BT Apply Changes

  • After the translation is done click on 'Import Patch' button at the bottom of the screen and choose the file from the file explorer.

  • File will be loaded on the right side of the screen.

  • Click on Apply patch. The right document will contain a detailed description if any error is encountered. Applying can sometimes take a while, user should not close the application while it is underway.

  • Patches that can be applied are applied, and failed patches are stored in Failed.sew in the target repository. A small description is given as to why the patch application may have failed. The user may use the hyperlink to open Failed.sew, make the necessary changes and import to file to apply the remaining changes.

  • Generally, “Error: Cannot Apply Patch” or “CouldNotApplyAllChanges” messages occur because you are trying to apply changes already there in the target repository.

  • Use the Restore Changes button to run the “git restore .” command (remove all un-staged changes) and the Force Clean button to run “git clean -f” command (remove any unwanted files) on the target repository.

  • If any patch is applied, a diff is shown in winmerge/any difftool, if it is configured.

  • Changes can also be seen in the tool’s right window by choosing files from the treeview on the left. Sew also supports applying changes made in other file formats such as .pngs.

Currently, only .png is supported for images.

  • Use the following Shortcuts to navigate in the Flow Documents:

    1. Press Ctrl+G to go to a specific line in document after content has loaded.

    2. Press Ctrl+F to find occurrences of words.
