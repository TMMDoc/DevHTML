Translation

TINC software products are translated into 23 international languages.

The following chart gives an overview of translation requirement for different Trumpf software products:

There are two translation activities handled at TINC.

  1. Translation of the UI elements used in the software supported by online Xlate tool.

  2. Translation of the bento documentation which has UI elements embedded in the documents.

These translations are triggered at three months intervals for the Xlate UI elements and at six months intervals for the bento documentation. Translation activity is outsourced to a translation agency — Seprotec.

Xlate tool is used to introduce citations in the bento documentation and these must exactly match the UI elements used in the actual software.

For any new software product created at TINC, the initial version of the document is created in English and the entire asciidoc files are sent to the translation agency for translation into all the supported languages. The translated asciidoc files are then stored in the corresponding GitHub documentation repositories language wise.

For the next translation cycle, the translation agency is provided with the diff between the initial version and the second version of the asciidoc files. This diff is extracted as a patch and the patch is send for translation. This saves the translation cost for the upcoming versions (by not translating the entire document every cycle). Once the patch is translated, it is inserted back into the second version of the corresponding language document repositories.

From there, the bento tool is used to generate HTML files and published.

The extraction and insertion of the patch is done using Book Tool.
