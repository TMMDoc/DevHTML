Flavoring

Flavoring is the process to modify a product to the identity of a sales channel (Trumpf, Lantek, JFY). Bento has a set of functionality to perform this tasks by conditional compilation. The flavoring is mainly defined by attributes set in the manifest.txt file.

Tip
 Refer to the Bento documentation for a general description and the command reference

A typical manifest.txt for a Metamation product looks like (shortened to the significant elements):

FLAVORS=FLUX, JFY, TRUMPF, BOOST   -> The flavor alternatives
ATTRIBUTE=APPNAME:Flux          -> Substitution Attributes
ATTRIBUTE=CUTAPP:Flux
ATTRIBUTE=BENDAPP:Flux
ATTRIBUTE=MIS:Praxis
#if TRUMPF | BOOST              -> if condition based on Flavor
ATTRIBUTE=APPNAME:TecZone       -> replace Attributes for this Flavor
ATTRIBUTE=CUTAPP:TecZone Laser
ATTRIBUTE=BENDAPP:TecZone Bend
ATTRIBUTE=MIS:Praxis
  #if BOOST
  ATTRIBUTE=integrated          -> Conditional Attributes
  #endif
#endif

FLAVORS

All possible flavors are the first line of manifest.txt.

FLAVORS is used for:

  • Control attributes within the manifest file itself

  • Add or exchange files

    Impressum.adoc             <-- Used for FLUX, TRUMPF flavors
JFY.Impressum.adoc         <-- Used only for JFY flavor

  • Control alternative bitmaps

    [img]
logo.png               <-- Used for FLUX flavor
TRUMPF.logo.png        <-- Used for TRUMPF flavor

  • Define an alternative CSS files

    JFY.Site.css          <-- Custom site CSS (framing style)

Substitution Attributes (ATTRIBUTE= …​ : …​ )

Within Metamation we call all technology modules Flux, Trumpf distinguishes the modules like TecZone Bend etc.

The Substitution Attributes replace the product name within ADoc files

Needed attributes for FluxBook:

  • APPNAME: the general product name for Flux

  • CUTAPP, BENDAPP, FOLDAPP: The specific module names for the technologies

  • MIS: The part management system like Praxis

Needed attributes for Praxis:

  • APPNAME: the general product name for Praxis

  • FLUXAPP: Branded name of Flux

  • FLUXDOCPATH: Path for referencing Flux

Conditional Attributes (ATTRIBUTE=…​)

Used for:

  • Control if documents or subfolders are considerd within contents.txt

    Installation        #ifndef INTEGRATED
[Settings]
PraxisLink          #ifdef FLUX

  • do conditional paragraphs in ADocs

    ifdef::INTEGRATED[]
The text here is included only in the INTEGRATED flavor of the product and describes how TecZone is called from HomeZone
endif::[]

Needed attributes for FluxBook:

  • INTEGRATED: removes chapters like installation or part loading in case Flux is integrated in Praxis

Needed attributes for Praxis:

  • none
