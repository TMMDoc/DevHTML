Documentation

The Documentation team provides support for the creation of product documentation.

  • Documentation requirements are defined based on the input from the technical experts (see roles).

  • The process follows a FogBugz driven workflow (see FogBugz).

  • For an overview of the complete documentation process, refer to the workflow summary (see Workflow).

  • The base documentation is translated (see Translation). With each new version, differences from the previous version must be identified. This is handled using the BookTool.

  • Relevant chapters from the FluxBook are compiled, and customer-specific theming is applied (see Flavoring).

  • Words or phrases may have multiple translation options. When referencing HMI text elements, it is essential to use the exact wording as shown in the HMI. This consistency is ensured through the use of Citations.

Hosting of the HTML documents

  1. The official released version of all the documents produced at TMM are listed below in Documentation Index topic.

  2. The Sharepoint site Manuals - Documents folder provides the latest content that reflects the main branch of a document.

  3. The Sharepoint site Manuals - Proofread folder provides full or fragments of documents to review.
