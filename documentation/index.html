Documentation

The Documentation team supports the creation of product documentation

  1. The Documentation needs is based on the input of the technical experts (see roles )

  2. its based on a FogbugZ workflow (see FogbugZ)

  3. See the complete workflow (see Workflow)

  4. The base documentation gets translated (see Translation). With each version we need to identify the difference between the new and the previous version. This is dode vith the BookTool.

  5. The relevant chapters of FluxBook are combined and a customer specific theming is added (Flavoring)

  6. A word or a sentence could often be translated in different ways. In case the documentaion refers to text elements of the HMI it is important to state exactly the same wording of the HMI. This is done by Citations

Hosting of the HTML documents

  1. The Render-online manual always provides the latest official released version of a Document

  2. The Sharepoint Manuals - Documents folder provides the latest main branch of a Document

  3. The Sharepoint Manuals - Proofread folder provides full or fragments of documents to proof read
