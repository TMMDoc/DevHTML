Fluxtester Repository Overview

Fluxtester is a UI test automation framework for the Flux application. Here’s what I found:

Core Details

  • Type: Console-based test automation runner for Windows WPF applications

  • .NET Target: .NET 10 (Windows platform)

  • Purpose: Automated testing of Flux UI with custom test fixtures and test cases

  • Entry Point: FluxTester.cs - Main console application

Key Features

  • Test Runner: Custom test execution engine with support for:

  • Test fixtures and individual tests

  • Options for regenerating expected outputs (-regenerate)

  • Test result comparison with Windiff (-diff)

  • Stop on failure mode (-stop)

  • Failed test re-runs (-failed)

  • Fixture listing (-fixtures)

  • Interactive Control: CapsLock key to stop tests during execution

  • Test Management: Automatic test/fixture ID tracking and sequencing (nextid command)

Dependencies

  • Flit.dll - Core testing framework library (external dependency from metamation/TestFramework project)

  • FlaUI.UIA3 (v3.2.0) - UI Automation framework for Windows accessibility

  • Custom utilities in Utils.cs for test data and operations

Architecture

  • Modular test structure with subdirectories (e.g., FluxCut/ for specific test suites)

  • Resource-based test data in Resources/TData/

  • Settings and design-time resources for WPF support

  • Unsafe code blocks enabled for advanced interop operations

This is a specialized test automation framework tailored for validating Flux application UI behavior with detailed reporting and flexible test execution options.
