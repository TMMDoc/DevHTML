Architecture & Patterns

Test Organization: - Uses [TestFixture] attributes with unique IDs (nested class structure) - Each fixture contains multiple [Test] methods with unique test IDs - Supports modules (e.g., "FluxCut", "Bend", "Databases") for filtering test execution - Tests marked with TestType (Functional, Smoke, etc.)

Key Utilities Provided: - UI automation via FlaUI - File/resource management - Flux process lifecycle control - Keyboard/mouse input simulation - Screenshot/comparison capabilities - Result tracking and HTML reporting

Test Execution Features: - CapsLock interrupt mechanism - -regenerate flag for updating test baselines - -diff for visual comparison with WinMerge - -failed to re-run failing tests - Module/subsystem filtering - Per-developer ID ranges to prevent conflicts

This is a specialized Windows UI test automation framework for validating the Flux application with organized test fixtures across multiple functional areas.
