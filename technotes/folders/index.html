Folders App

This workspace is organized into a small set of operational areas:

Top-Level Folders

  • Archive/

    Legacy or retained data kept for reference.

  • BendTools/

    Tooling and supporting resources for bend operations, including manifests, versioned content, and list data.

  • Machine/

    Machine-specific configuration and data folders, typically organized by machine ID. These contain files such as machine.curl, cut database files, settings, and usage-related text files.

  • Output/

    Generated artifacts from processing runs.

    Contains:

    • Code/ for generated code and list-style output files such as .jupidu, .LST, and .fx

    • Report/ for generated HTML reports that correspond to processed jobs

  • Recent/

    Recently used or transient working data, including notes and runtime support files.

  • Samples/

    Sample resources, including report templates or example output structures.

  • System/

    Shared system-level configuration and database description files used by the application.
