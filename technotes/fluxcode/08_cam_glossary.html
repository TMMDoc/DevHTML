CAM Glossary with Code Mapping

CAM Glossary with Code Mapping

Goal: a single, alphabetized reference of every CAM and Flux-internal term you’ll encounter in the first six months, with a one-line definition and a code or doc location. Use this as a lookup, not a read-through.

A

ACB — Adjustable Crowning Beam Mechanical compensation for press-brake bed deflection under load. Code: Src\Core\Bend\BendACB.cs. Image: Doc\Img\Bend\deflection.png. Note: not the same as the ACB laser (a sensor on some Trumpf brakes).

Air bend A bend where the punch does not bottom out in the die — the most common bend kind. Code: EBendKind.AirBend in BendCore.cs.

Air gap The intentional gap between the upper blade and the sheet on a panel bender. Doc: Notes\Tech\2018.AirGapTables.ftx.

Au / Gold Flux’s compact binary serialization format. Doc: Notes\Dev\1003.AuGoldEncoding.ftx.

Auto-tooler The engine that turns a BendTech into a sequence + setups + BG positions. Code: Src\Core\Bend\Ops\BendTooler.cs.

B

B-Rep Boundary representation of a 3D model. Flux uses a mix of B-Rep and meshes.

BA — Bending aid Helper that supports the sheet during bending. Has a position and an enabled state per op. Code: Src\Core\Bend\BendAid.cs, Ops\BAPlacer.cs. Image: Doc\Img\Bend\BendAidLiftPos.png.

BCell Bend Cell — automated bending with a robot, gripper, suction cups, regrip stations. Code: Src\Core\BCell\ (34 files), Src\UI\BCell\.

BendOp A single bending operation on a BendTech. Sealed partial class with ~200 properties. Code: Src\Core\Bend\BendOp.cs.

BendSetup A complete tool layout on the press brake. Code: Src\Core\Bend\BendSetup.cs.

BendTech The press-brake projection of a Part. Code: Src\Core\Bend\BendTech.cs.

BG — Back gauge The rear stop the operator pushes the sheet against to position it. Code: Src\Core\Bend\BendBG.cs, BGPath.cs, Ops\BGPlacer.cs. Image: Doc\Img\Bend\BGFlexEdge.png.

Blade On a panel bender, the moving upper / lower blade that forms the bend. Code: Src\Core\Fold\ (search for Blade). Images: Doc\Img\Fold\Blade*.png.

BMC Trumpf machine format for press brakes. Code: Src\Core\Bend\Ops\ImportBMC.cs, ExportBMC.cs. Doc: Notes\Tech\2016.ImportBMC.ftx.

Boost Trumpf platform. Flux integrates as the Boost flavor. Doc: Notes\Tech\2012.BoostInterface.ftx.

Bump bend Many small bends to approximate a large radius. Code: EBendKind.BumpBend, BumpSteps on BendOp. Image: Doc\Img\Bend\BumpBendSpine.png.

C

Coining Bending where the punch is forced into the die enough to "set" the material — higher force, less springback. Code: EBendKind.Coining in BendCore.cs.

Composite glob The auto-synthesized controller when the user selects multiple objects of the same kind. Code: BendOpsGlob in Src\UI\Bend\BendOpUI.cs line 661.

CMD file A batch script that drives Flux operations. Doc: Notes\Tech\2005.FluxCommandFiles.ftx.

Crash / collision Detected interference between tools, sheet, and machine. Code: Src\Core\Bend\BendOpCrash.cs (classic) and NBendCrash.cs (new).

Curl Flux’s text-based serialization format (Lisp-flavored). Doc: Notes\Tech\2007.CurlFiles.ftx.

Cup gripper A vacuum-cup gripper used by the bend cell. Code: Src\Core\BCell\ (search for Gripper). Doc: Notes\Tech\2017.CupGripperInfoInDwg.ftx.

D

Deduction (bend deduction) The amount by which the unfolded length is less than the sum of the flange lengths. Material-, thickness-, and tool-dependent. Code: BendStaples.Deduction in BendCalc.cs.

Determinism Property that the same inputs always produce the same outputs. Critical for golden-file tests; uses GeomSort instead of plain OrderBy. Code: Src\Base\Geo\GeomSort.cs.

Die The lower tool on a press brake. The sheet is bent into its V-groove. Code: BDie in BendTool.cs.

Die V-width The opening width of the die. Code: property on BDie. Image: Doc\Img\Bend\VWidthType.png.

DLD Trumpf NC format for press brakes. Code: Src\Core\Bend\Ops\DLDExporter.cs.

Dump The text-output method used by golden-file tests. Not a debug print; it’s the test contract. Code: BendSetup.Dump, BendTech.Dump.

Dwg The 2D drawing held on Part.Dwg. Contains E2* entities. Code: Src\Core\Drawing\.

E

E2* (E2Line, E2Arc, …​) Two-dimensional drawing entities. Code: Src\Core\Drawing\.

E3* (E3Plane, E3Flex, …​) Three-dimensional model entities. Code: Src\Core\Model\.

EBendKind The enum identifying the kind of bend (AirBend, OpenHem, ClosedHem, Coining, BumpBend, ZBend*, FlatBend, …​). Code: Src\Core\Bend\BendCore.cs.

Engine flavor Headless Flux variant embedded in Praxis. Doc: Notes\Tech\2013.FluxFlavors.ftx.

EResult The standard return enum for fallible operations (OK, plus failure modes). Code: Flux.Base.

EVF tool Extended-V tool for press brakes. Image: Doc\Img\Bend\EVF-tool.png.

F

Feature bit A flag controlling which Flux features are active in a given flavor. Doc: Notes\Tech\2013.FluxFlavors.ftx lines 38—​57.

FlatTech (= CutTech) The flat-cutting projection of a Part (laser / punch / combo). Code: Src\Core\Laser\, Src\Core\Punch\. Held on Part: Part.CutTech.

Flavor A build variant of Flux (Dev, Flux, Trumpf, Boost, Engine, B23, Fold, Tech). Doc: Notes\Tech\2013.FluxFlavors.ftx.

Flex A bend zone in the 3D model — E3Flex. Distinct from BendOp (which represents the bending operation). Code: Src\Core\Model\E3Flex.cs.

Flit The Flux test framework. [TestFixture] / [Test] attributes; namespace Flit. Code: Test\ with namespace Flit;.

Forming A non-bending feature (louver, lance, rib). Code: Src\Core\Model\Forming.cs.

FoldTech The panel-bender projection of a Part (TecZoneFold). Code: Src\Core\Fold\.

FX file The native Flux save format. Zip of Curl + Au + thumbnail.

FX (helper) The static helper class in Flux.Base (logging, test helpers, FX.Notes storage). Code: Src\Base\ (search class FX).

G

GEO Trumpf’s geometry exchange format. Common test-input format. Code: Src\Core\IO\GeoReader.cs, GeoWriter.cs.

GElement Abstract description of a glob property (UI-agnostic). Code: Src\UI\ (search class GElement).

Glob A controller wrapping a single domain object for UI editing. Composite globs handle multi-selection. Code: Src\UI\Glob.cs. Examples: BendOpGlob, LaserCutGlob, BCellGlob.

Gold (= Au) Same as Au.

Golden file The reference .txt file that a test’s Dump output is compared against. Doc: chapter 06.

Group bend Multiple parallel flexes performed in one bending operation. Code: BendTech.cs constructor (lines 49—​80).

H

Hem A seam-style bend. Open hem leaves a gap; closed hem closes it. Code: EBendKind.OpenHem, .ClosedHem, .OpenHemPrebend, .ClosedHemPrebend in BendCore.cs.

Hex A small Flux-internal scripting / interpreter library (Src\Base\HexLib.cs). Originally (Note 2004, 2016) planned as the post-processor language; that role was superseded by C# writers in Src\API\Bend\Post\. Today Hex survives at most as legacy / scripting infrastructure. Note 2004 is historical context, not a current spec.

Hoops TechSoft 3D library used for CAD-format import. Wrapped by Flux.Hula64.dll.

Holder A tool holder (adapter, extender, radius holder) that mounts another tool. Code: BHolder in BendTool.cs.

Hub Static class providing typed delegate slots for cross-layer wiring (App ↔ Core/UI). Code: Src\Core\Hub.cs.

Hula / Flux.Hula64 The C++/CLI wrapper around HOOPS, providing 3D import. Code: Src\Hula\. Build: see Notes\Dev\1002.BuildingFlux.ftx.

I

IBendCalc Interface for the bend-physics calculator. Code: Src\Core\Bend\BendCalc.cs.

IFlexSpine Interface for a bend cross-section. Code: Src\Core\Model\BendSpine.cs.

INotifyPropChanged Flux’s variant of property-change notification (used jointly with undo). Code: Flux.Base.

ISheafSite Marker interface for objects whose settings cascade up to a parent. Code: Src\Core\Settings\Sheaf.cs.

J

Jaw gripper A mechanical gripper variant for the bend cell. Code: Src\Core\BCell\ (search BJaw*).

K

KB40 A Trumpf folding-machine variant. Mentioned in air-gap tables and tooling. Image: Doc\Img\Fold\KB40MaxAdjust.png.

Kerf Material removed by a laser cut (the cut width).

L

Layer Render layer (ELayer.BendMount, ELayer.LaserCut, …​) controlling drawing visibility and order. Code: Src\Base\Lux\Layers.cs.

Layer2 The drawing’s 2D layer system (Dwg.Layers). Code: Src\Core\Drawing\.

Leeward The "up" side of a bend (closer to the base plane in the flat-bend tree). Doc: Notes\Dev\1005.BendAreaModeling.ftx.

Lux The OpenGL rendering layer (in Flux.Base). Code: Src\Base\Lux\.

M

Machine Abstract base for per-domain machine definitions. Loaded from X:\Data\Machine\<id>\. Code: Src\Core\ (search class Machine).

Make The conventional name for static factories (BendTech.Make, Machine.Get).

Mech A 3D animatable model of a machine. File: Machine.mech per machine folder.

MetaCAM / Trek Legacy CAM stack. Trek is a private fork shipped with Flux as a CLI for legacy operations. Doc: Notes\Tech\2011.MetaCAMInterface.ftx.

Microjoint A small unbroken connection left in a laser-cut profile so the part doesn’t drop out before unloading. Image: Doc\Img\Laser\MicrojointLength.png.

Mood A render-style scope (EMood.Selected, Normal, …​). Code: Src\Base\Lux\Lux.cs.

Mount A tool placed at an X position on the brake table. Code: BendMount in Src\Core\Bend\BendMount.cs.

N

Nesting Arranging multiple parts on a sheet to maximize material usage. Code: Src\Core\Nest\.

NBendCrash The "new" collision engine for bend operations, parallel to BendOpCrash. Code: Src\Core\Bend\NBendCrash.cs.

NC code Numerical control code emitted to the machine. For press brakes it is generated by a C# post-writer in Src\API\Bend\Post\ (TrumpfDLD, Bystronic, LVD, BNCWriter for Cell5K / TrumpfB23, …​) dispatched via Hub.GenerateNC. The legacy Hex / PDL languages are no longer used at runtime.

O

OEM OEM-specific customizations. Code: Src\Core\OEM\, Src\API\OEM\.

Op Shorthand for an operation (BendOp, LaserCut, etc.).

Open height / closed height The press brake’s vertical machine parameters defining the stroke. Code: BendMachine.OpenHeight, .ClosedHeight.

P

Part Top-level job aggregate. One Part per opened document. Code: Src\Core\Part\Part.cs.

Phase A subdivision of a bend (e.g. hem prebend → final hem). One flex can have multiple phases. Code: BendOp.Phases in BendOp.cs.

Pline The polyline workhorse type. Code: Src\Base\Pline\, Src\Base\Geo\Profile.cs.

PostLoad Attribute on a method that runs after Au-deserialization, used for forward migration. Code: search [PostLoad] (e.g. BendSetup.cs lines 64—​76).

Praxis Internal Trumpf system embedding Flux as the Engine flavor. Doc: Notes\Tech\2013.FluxFlavors.ftx.

PDL Legacy post-processor language. Effectively deprecated; current postprocessors are C# in Src\API\Bend\Post\.

Post writer (NC) A C# class in Src\API\Bend\Post\ that emits NC code for a specific machine dialect (TrumpfDLD, Bystronic, LVD, Cybelec, Safan, Cell5K/B23 via BNCWriter, …​). Routed via Hub.GenerateNC → Src\API\Basis\Basis.cs::GenerateNC (line 138). The dispatch key is the basename of BMachine.Post; the legacy .hx extension on that filename is a routing token only.

Punch The upper tool on a press brake; or the cutting domain for sheet metal. Code: BPunch in BendTool.cs; domain at Src\Core\Punch\.

Q

(reserved)

R

Rear stop / back gauge — see BG.

Regrip The bend cell’s gripper releases and re-grips the part mid-job, often at a regrip station. Code: BCRegrip in Src\Core\BCell\. Image: Doc\Img\BCell\Slide.png.

Robotic bend A bend cycle performed in a bend cell with a robot. Code: BendOp.Robotic and the RegripDuringBend family of properties.

S

Setup A tool layout on the machine. One press brake job often has 1—​3 setups. Code: BendSetup in Src\Core\Bend\BendSetup.cs.

Sheaf The cascaded settings dictionary. Every ISheafSite has its own dict and a parent. Code: Src\Core\Settings\Sheaf.cs.

Spine The center-line through the cross-section of a bending zone. Code: IFlexSpine in Src\Core\Model\BendSpine.cs. Doc: Notes\Dev\1005.BendAreaModeling.ftx.

Springback The angular rebound of material after the punch retracts. Code: BendStaples.SpringbackD in BendCalc.cs.

Station / stage An X region of the brake table that holds an independent set of tools. Code: inside BendSetup.

STL Mesh exchange format. Code: Src\Core\IO\STLModel.cs.

T

TBS — Trumpf Bend Solution Trumpf’s own bend-physics calculator. Code: TBSBendCalc.cs, TBSCustomDB.cs.

TecZoneBend / TecZoneFold Trumpf brand names for the press-brake / panel-bender flavors.

Tech Domain projection of a Part (BendTech, FoldTech, FlatTech, …​).

TLM / TLE — Trumpf License Manager / Expert Two licensing backends. TLM (Sentinel) for Fold flavor; TLE (WIBU) for Trumpf and Boost. Doc: Notes\Tech\2013.FluxFlavors.ftx.

Trek The private fork of MetaCAM shipped with Flux as a CLI. Doc: Notes\Dev\1004.TrekCommandLine.ftx.

TSO — Tool Setup Optimizer Optimizer for tool layouts across a sequence. Code: Src\Core\Bend\TSOBatch.cs, Src\Core\Bend\Ops\BOptimizer.cs.

U

UID A per-tech unique identifier (e.g. BendOp.UID, BendSetup.UID). Stable across save/load.

Undo Per-part undo stack. Mutations registered through BA.Set are automatically tracked. Code: Src\Base\Undo\.

V

Valency The mounting compatibility "type" between tools and adapters. Code: BTool.TopValency, BTool.BottomValency.

VDwg Vector-drawing format used to render geometric images. Doc: Notes\Dev\1013.TheVdwgFileFormat.ftx.

VModel Base class for view-models. Lives next to its model in Flux.Core. Code: Src\Base\Lux\VModel.cs.

V-width — see Die V-width.

W

Wad / Flux.wad Zip of read-only resources (shaders, metadata, fonts). Doc: Notes\Dev\1002.BuildingFlux.ftx lines 223—​238.

Waypoint A discrete robot-cell position in the trajectory of a bending job. Code: Src\Core\BCell\ (search Waypoint).

Windward The "down" side of a bend (further from the base plane). Doc: Notes\Dev\1005.BendAreaModeling.ftx.

Workpiece Record bundling material + thickness for bend calculations. Code: BendCalc.cs.

X / Y / Z

XVec / YVec / ZVec The local coordinate system axes on an E3Plane. Doc: Notes\Dev\1005.BendAreaModeling.ftx lines 91—​105.

Z-bend A bend where two close opposing flexes form an offset. Code: EBendKind.ZBendAirBend, .ZBendCoining. Image: Doc\Img\Bend\ZBendTool.png.
