Guided Code Tours

Goal: seven structured, time-boxed reading walks through the most important Flux flows. Each tour starts at a precise file and line, has named stops, and ends with a question to confirm understanding. Every tour is doable read-only — you do not need to compile or run anything.

Use this when: you want to build mental scaffolding for an unfamiliar area in 15—​20 minutes flat.

How to use these tours

  • Open exactly the files named. Don’t wander; you can always do a second pass.

  • Don’t read the whole file. Stop at the named lines.

  • At each stop, write one sentence in your own words about what you just read. If you can’t, re-read.

  • End with the self-check question. If you can’t answer, redo the last stop.

How to read these tours productively

  • Time-box each tour to its target. Going long means you’re trying to memorize, not understand.

  • Repeat tours weekly for the first month. The same tour reveals more on the second pass.

  • Make your own notes. Three lines per tour is plenty.

  • Discuss with your buddy. Each tour ends with a question — answer it out loud.

A gentle calendar for the first three weeks

Week Tours to do

1

2

B, C, E — Glob, rendering, persistence.

3

F, H — bug fix in the wild, layer wiring.

By the end of week 3 you’ve internalized the full reading vocabulary of the codebase.
