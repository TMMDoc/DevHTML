Part.Dwg

Part.Dwg

Part.Dwg holds the complete 2D representation of a sheet-metal part after it has been unfolded from its 3D model (or imported directly from a flat format such as DXF). It is an instance of Drawing2 — a container that stores every geometric and semantic element of the flat blank.

File: Core/Drawing/Drawing.cs — class Drawing2

The Big Picture

Drawing2
 ├── AList<Layer2>     Layers      ← named groups with display and semantic roles
 ├── AList<Entity2>    Entities    ← all geometry and annotation
 │    ├── E2Pline[]               ← contours  (blank outline, holes, slots …)
 │    ├── E2Bendline[]            ← bend marks (where to fold, at what angle)
 │    ├── E2Point[]               ← pierce points, forming centres, approach markers
 │    ├── E2Text[]                ← text labels, part numbers, engrave text
 │    ├── E2Insert[]              ← block references (DXF blocks / sub-assemblies)
 │    ├── E2Spline[]              ← free-form spline curves (from DXF)
 │    └── E2Dimension[]           ← dimension annotations
 ├── List<Block2>      Blocks      ← reusable block definitions (referenced by E2Insert)
 └── List<FormInfo>    FormInfos   ← 3D press-form metadata (from .geo import)

Every entity knows which Layer2 it lives on. The layer carries both display information (colour, line type, visibility) and a semantic role (EUse) that tells the rest of the system how to treat the entity — whether to cut it, mark it, skip it for nesting, etc.

E2Pline — Contours

File: Core/Drawing/2DEntity.cs — class E2Pline

Line and arc segments that make up the contour

IsClosed

bool

Whether the contour is a closed loop

bool

Bound

Bound2

Axis-aligned bounding box

Area

double

Enclosed area (positive; holes subtract from the part area)

How Inner and Outer Is Determined

Drawing2.PrepareForCAM() calls DoQuickMarkInOut() (or DoSlowMarkInOut() for complex drawings). The algorithm sorts all closed plines by descending area and marks a pline as IsOuter = true if it is not contained inside any other outer pline.

The largest closed pline that is not contained inside another closed pline becomes the outer blank boundary. Every closed pline that lies inside the outer boundary is a hole (IsOuter = false).

What Contours Are in the Drawing

Contour Origin

Outer blank boundary

Unfolded from E3Plane.Contours via Unfolder.AddPlaneLoops

Holes and cutouts

Unfolded from inner contours of E3Plane faces, or imported from DXF

Open paths (slots, etch)

Imported from DXF, or from markings on the 3D model

Approach paths

Added by LaserAutotooler on layer CutTooling

How Bend-Line Points Work

Pts always has an even number of entries. Each pair [i, i+1] is one contiguous segment of the bend line. A bend line that is interrupted by a hole will have multiple segments (4 or 6 points). The segments are always collinear — they all lie on the same straight line.

// Iterate bend-line segments:
for (int i = 0; i < bendline.Pts.Length; i += 2) {
    Point2 start = bendline.Pts[i];
    Point2 end   = bendline.Pts[i + 1];
    // draw or process this segment
}

Connection Back to the 3D Model

Every E2Bendline has the same UID as the E3Flex that produced it during unfolding. This lets the system navigate in both directions:

// From drawing → 3D model:
E3Flex flex = bendline.PairedFlex;     // looks up by UID in Part.Model

// From 3D model → drawing:
E2Bendline bl = drawing.Bendlines
    .FirstOrDefault(b => b.UID == flex.BendlineUID);

Internal State Flags (EFlags)

Drawing2 tracks its own preparation state with an internal EFlags bitfield. These flags prevent redundant work when CAM algorithms call PrepareForCAM multiple times:

Flag Meaning

OrientedCCW

All closed plines have been oriented counter-clockwise

InOutMarked

IsOuter has been computed for all plines

CleanedPlines

Duplicate nodes and collinear segments have been removed

SingleOuter

There is exactly one outer pline (fast path for tooling)

BendSequenced

Bend-lines carry a valid SeqNo processing order

BendAnnotated

Bend-lines have been updated with punch/die names from BendTech

FoldAnnotated

Bend-lines have been updated with air-gap from FoldTech

PreprocessForCAM

Counter-sinks identified, text converted to plines

PrepareForCAM — Making the Drawing Ready for Tooling

Before laser tooling (or any CAM operation) reads the drawing, PrepareForCAM() is called. It runs a sequence of normalisation steps, each guarded by a flag so it only runs once:

PrepareForCAM()
 ├── DoCleanupPlines()         — remove degenerate segments, collinear nodes
 ├── SplitCircleToArcInOpenPlines() — convert full circles in open plines to two arcs
 ├── DoOrientCCW()             — ensure all closed plines wind counter-clockwise
 ├── DoQuickMarkInOut()        — mark IsOuter for simple drawings (one outer pline)
 │    └─ DoSlowMarkInOut()     — fallback for multi-outer drawings
 └── DoCAMPreprocessing()
      ├── Identify counter-sinks (concentric circles) → move outer to FormFootprint layer
      └── Convert E2Text on cut layers → E2Pline (so they can be laser-engraved)

Complete Memory Layout

Drawing2
 │
 ├── Layers: [Layer2, Layer2, …]
 │    each: Name, Color, LineType, EUse (Std/Aux/Mark/CutTooling/…), Visible, Locked
 │
 ├── Entities: [Entity2, Entity2, …]
 │
 │   ┌─ E2Pline ──────────────────────────────────────────────────────────┐
 │   │  Layer → Layer2 (e.g. Std = to be cut)                             │
 │   │  Flags → E2Flags.BevelCut, SkipLaserTooling, …                    │
 │   │  UID   → stable int used by tooling to find its cut steps          │
 │   │  Pline → Segment[] (lines + arcs)                                  │
 │   │           IsOuter, IsClosed, Bound, Area                           │
 │   └────────────────────────────────────────────────────────────────────┘
 │
 │   ┌─ E2Bendline ───────────────────────────────────────────────────────┐
 │   │  Pts[]    → pairs of Point2 (each pair = one bend fragment)        │
 │   │  Angle    → exterior bend angle (radians, ± = valley/mountain)     │
 │   │  Radius   → inner bend radius (mm)                                 │
 │   │  Thickness → sheet thickness                                       │
 │   │  KFactor  → neutral-fibre ratio (default 0.35)                     │
 │   │  FlatWidth → developed width of bent zone                          │
 │   │  Deduction → bend deduction                                        │
 │   │  SeqNo    → processing order (1 = first)                           │
 │   │  Kind     → Std / Coining / Hem / ZBend / …                        │
 │   │  PunchName, DieName → set by BendTech after tooling                │
 │   │  Airgap   → set by FoldTech after tooling                          │
 │   │  UID      → same as paired E3Flex.BendlineUID in the 3D model      │
 │   └────────────────────────────────────────────────────────────────────┘
 │
 │   ┌─ E2Point ──────────────────────────────────────────────────────────┐
 │   │  Pt    → Point2 coordinate                                         │
 │   │  Layer → ApproachMarker / FormCenter / CutTooling / …              │
 │   └────────────────────────────────────────────────────────────────────┘
 │
 │   ┌─ E2Text ───────────────────────────────────────────────────────────┐
 │   │  Text, Pt, Height, Fontname, Angle, Alignment                      │
 │   │  Converted to E2Pline during PrepareForCAM if on a cut layer       │
 │   └────────────────────────────────────────────────────────────────────┘
 │
 │   ┌─ E2Insert ─────────────────────────────────────────────────────────┐
 │   │  Block → Block2 (look up by name from Drawing2.Blocks)             │
 │   │  Pt, Angle, XScale, YScale → placement transform                   │
 │   └────────────────────────────────────────────────────────────────────┘
 │
 │   ┌─ E2Spline ─────────────────────────────────────────────────────────┐
 │   │  Spline → Spline2 (NURBS / B-spline from DXF)                      │
 │   └────────────────────────────────────────────────────────────────────┘
 │
 ├── Blocks: [Block2, …]
 │    each: Name, Base (Point2), Entities (List<Entity2>)
 │
 └── FormInfos: [FormInfo, …]
      each: Position, Angle, Mesh, PunchName, DieName, OnBackFace, DuringBending
