Part

Part

Everything in Flux revolves around a single central concept: the Part. A Part is the digital twin of a physical sheet-metal component — it holds the 3D geometry, the 2D flat drawing, and all the manufacturing instructions needed to produce that component on real machines.

Every other major object either belongs to a Part or organises Parts for production.

Part                       ← one sheet-metal component
 ├── Model3                ← its 3D folded shape
 ├── Drawing2              ← its unfolded flat shape
 ├── BendTech              ← instructions for a press-brake
 ├── FoldTech              ← instructions for a panel-bending machine
 ├── FlatTech (CutTech)   ← instructions for laser / punch cutting
 ├── Layout                ← single-part arrangement on one sheet
 └── TrimJob / TrimTech   ← instructions for 5-axis trimming

At a higher level, many Parts can be grouped together for production:

NestJob                    ← one production batch
 ├── NestPart[]            ← which parts, how many
 ├── NestSheet[]           ← which raw-material sheets are available
 └── NestPattern[]         ← calculated sheet arrangements (results)

Active Part

Part.Current is the static property that holds whichever Part the user is currently working on. This is the application’s global selection state — think of it as "the open document".

Part active = Part.Current;   // the part currently open in the editor

3D Geometry — Model3

File: Core/Model/Model3.cs

Model3 represents the part in its folded, three-dimensional state — what you would hold in your hands.

It stores a tree of connected planar faces and bends:

Property Type Meaning

Entities

AList<Entity3>

All geometric elements (faces, bends, meshes)

Baseplane

E3Plane

The reference face — root of the geometry tree

Flexes

IEnumerable<E3Flex>

All bend lines in the model

FlatMesh

CMesh

Triangle mesh in the fully flat state

FoldedMesh

CMesh

Triangle mesh in the folded state

Category

ECategory

Classification (sheet metal, laser, assembly …)

MassProps

MassProps

Centre of gravity, volume, weight

Entity Hierarchy

Entity3 is the base class for all geometric elements inside a model:

Entity3                 ← abstract base (has a name, transform)
 ├── E3Thick            ← has a material thickness
 │    ├── E3Plane       ← a flat face (most common element)
 │    └── E3Flex        ← a bend connecting two faces
 ├── E3Surface          ← imported freeform surface
 ├── E3Insert           ← assembly sub-component
 └── E3Mesh             ← arbitrary triangle mesh

E3Plane objects are the faces of the part. E3Flex objects are the bends that connect those faces. Together they form a directed acyclic graph (DAG) rooted at Baseplane:

Baseplane (E3Plane)
 └── [via E3Flex "bend A"]
      └── Side face (E3Plane)
           └── [via E3Flex "bend B"]
                └── Flange (E3Plane)

Every E3Thick has Windward (parent) and Leeward (child) pointers to navigate this tree.

2D Flat Drawing — Drawing2

File: Core/Model/Drawing2.cs

Drawing2 is the unfolded, two-dimensional representation — the flat blank that will be cut from a sheet.

Property Type Meaning

Entities

Entity2[]

All 2D elements

Plines

E2Pline[]

Contours (outer edge, holes, internal features)

Bendlines

E2Bendline[]

Lines where the part will be bent

The most important element is E2Pline — a 2D polyline that describes a contour. The outer boundary of the flat part is one E2Pline; every hole or cutout is another.

Polylines — Pline

File: Base/Pline/Pline.cs

Pline is one of the most frequently used types in the codebase. It represents a 2D contour made of straight lines and arcs.

Property Type Meaning

Segs

The individual line or arc segments

bool

Whether the contour is closed (a loop)

Think of Pline as the 2D equivalent of a shape — it can be the outline of the whole part, a rectangular hole, or an arc slot.

Bend Operations — BendOp

Each BendOp is one individual bend, in sequence:

Property Type Meaning

Flex

E3Flex

The model element being bent

EBendKind

What kind of bend (air bend, hem, coin …)

double

Target bend angle in degrees

How the operator re-orients the part after this step

Tools

BendToolSet

Punch and die selected for this step

EBendKind — Bend Types

enum EBendKind {
    AirBend,        // most common: punch doesn't reach die bottom
    Coining,        // high-pressure: punch presses all the way
    ClosedHem,      // folds flat onto itself
    OpenHem,        // folds back but leaves a gap
    BumpBend,       // multiple small hits to form a curve
    FormBend,       // uses a special forming tool
    // … and several pre-bend variants
}

EHandlingFlip — Part Re-orientation

After each bend the operator (or robot) may need to turn the part:

enum EHandlingFlip {
    None,       // keep orientation
    Bump,       // push forward along the machine
    RotateL,    // rotate left 90°
    RotateR,    // rotate right 90°
    Rotate180,  // rotate 180°
    Flip,       // flip over (turn upside down)
}

Nesting — NestJob

File: Core/Nest/NestJob.cs

Robotic Bend Cell — BCellTech

File: Core/BCell/

When a press-brake is combined with a robot for automated material handling, the robotic cell configuration is stored in BCellTech.

The cell is composed of typed equipment items:

BCellItem               ← abstract: has a name, size, and position
 ├── BStorage           ← pallet, basket, conveyor (where parts wait)
 ├── BGripper           ← end-of-arm tool (vacuum cups or jaw gripper)
 ├── BLightBox          ← vision reference target
 ├── BFurniture         ← passive collision geometry (tables, fences)
 └── BEquipment         ← active conveyors

Key planning objects:

Type Meaning

BRobotPose

A robot TCP (tool-centre-point) position and orientation

BPickupPlan

How and from where the robot picks up a part

BDepositPlan

Where and how the robot places a finished part

BRegrip

A robot manoeuvre to re-orient the part mid-sequence

5-Axis Trimming — TrimJob / TrimTech

Files: Core/Trim/

For parts that need 5-axis cutting (e.g. pipes or complex 3D profiles), TrimJob and TrimTech fill the same role as NestJob and FlatTech do for flat cutting.

Type Meaning

TrimJob

The production job: which machine, which parts, quantities

TrimTech

CAM data for a single part: tool paths, lead-ins, sequences

Global State — FX

File: Core/FX.cs

FX is a large static class that acts as the application’s global state and service locator. A few key properties beginners will encounter frequently:

Property Meaning

FX.Part.Current

The Part currently open in the editor

FX.Settings

Application-wide user settings

FX.DefaultUnit

Whether the app uses millimetres or inches

FX.DataDir

Path to the installation data directory

FX.AdminMode

Whether admin-only features are enabled
Note
 FX extends BA (the Base utility module), which means all low-level helpers (logging, geometry, colours) are also accessible through FX.

Change Notifications

The data model uses a lightweight property-watching system instead of (or alongside) standard .NET INotifyPropertyChanged.

Classes that participate implement ISheafSite and expose PWatch / PUnwatch methods. Change events are identified by the EProp enum:

// Subscribe to changes on a part:
part.PWatch(EProp.DwgChanged, OnDwgChanged);

// Unsubscribe:
part.PUnwatch(EProp.DwgChanged, OnDwgChanged);

Common EProp values: DwgChanged, Bendlines, Visual, SheetSize, SheetGrain.
