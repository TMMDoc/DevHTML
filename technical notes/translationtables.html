Translation tables

Translation tables

Translation tables are managed in Flux using XLate tool. XLate tool source can be found in X:\Tools\XLate.

XLate is used to create xls files for all languages based on the .txt files in x:\wad\lang which are then sent for translation.

Once the updated/translated xls files are received, XLate is used to update the .txt files in x:\wad\lang back.

  • The XLATE MAKE ALL is used to generate updated translation spreadsheets for all languages. The existing translation spreadsheets are found in Y:/XLate. These are edited by this command, and any newly added strings (from the various text files are added in, with a blank space left for the translation; the blank cell is colored yellow to indicate that it must be filled). For all strings that already exist in the spreadsheet, the previous value is carried over. Thus, this incremental process creates a new spreadsheet with a few more rows added with English text (and the corresponding foreign text cell colored yellow). The help images are also inserted into the spreadsheet at this time. The new xls files created will be found in the Y:\XLate\New folder. Sample xls shown below:

All the languages will have English, DE and the respective language columns as shown above. Initially only the DE.xls is sent for translation. The updated DE.xls file is copied to Y:\XLate and then other language xls files are created to get the updated DE column. Probably this is to help the translators who can use either an English or German string to translate it to another language.

  • Once the updated translation spreadsheets are received back from the translation agencies, we copy these back into Y:/XLate and use XLATE USE ALL to generate fresh string-tables that use the newly translated phrases. This updates the .txt files in x:\wad\lang folder.

  • We use XLATE DELPICS ALL to remove all the help images from the translation spreadsheets before archiving them in the repository.

XLATE MAKE <XX> will make the xls for a particular language. Similarly, XLATE USE <XX> to update the .txt file back in x:\wad\lang folder. For eg: XLATE MAKE RU will create RU.xls in Y:\XLate\New folder. XLATE USE RU will update the RU.txt in lang folder.

  • When you add a new entry in string table, make sure you create an empty text file for that entry in the respective language folder when you use XLATE MAKE or XLATE USE. Refer Case 34012.
